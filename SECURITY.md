# Security Policy

## Reporting Vulnerabilities

Please report security vulnerabilities to: bryankaufman@mac.com

**Do not create public GitHub issues for security vulnerabilities.**

We will respond within 48 hours and work with you to understand and address the issue.

## HIPAA Compliance

This software handles Protected Health Information (PHI). All contributors must follow HIPAA guidelines:

- Never commit PHI to the repository
- Use anonymized or synthetic test data only
- Follow secure coding practices
- Report any potential PHI exposure immediately

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | :white_check_mark: |

## Security Best Practices

- Keep dependencies up to date
- Use environment variables for sensitive configuration
- Enable two-factor authentication on GitHub
- Review all pull requests for security implications
